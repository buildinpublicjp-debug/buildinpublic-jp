## Root Cause
`computeCameraPose()` had incorrect "up" vector calculation:
```
_up.set(-_normal.z, _normal.x, _normal.y)
```
This ad-hoc permutation doesn't produce a proper tangent vector for all ECEF positions. Camera orientation became unstable when flying to different lat/lng coordinates.

## Fixed
- `components/viewport/CityScene.tsx`: Replaced ad-hoc up vector with proper geodetic north calculation
  - east = cross(Z_axis, surface_normal)
  - north = cross(surface_normal, east)
  - Camera "up" = north direction at the target point
  - This is mathematically correct for any point on the WGS84 ellipsoid
